## 二、优化 TODO 清单

### 高优先级（工作量小、收益大）

- [ ] **API 版本化** — 将外部 API 从 `/api/external/*` 改为 `/api/v1/external/*`，避免未来破坏性变更影响所有客户端
- [ ] **安全响应头中间件** — 添加 CSP、X-Frame-Options、X-Content-Type-Options、HSTS 等安全头
- [ ] **自动依赖漏洞扫描** — CI 中集成 `pip-audit` 或 `safety`，或启用 GitHub Dependabot

### 中优先级（工作量中等、长期受益）

- [ ] **设置页面整体 UI 重构 — Tab 化 + 临时邮箱配置区分离**（TODO: `docs/TODO/2026-04-03-设置页面UI重构-临时邮箱配置区分离TODO.md` | PRD: `docs/PRD/2026-04-04-设置页面UI重构-临时邮箱配置区分离PRD.md`）
  - 当前设置页为单一长列表，9 个区块无分类，UX 差
  - **Tab 结构（4 Tab）**：基础（登录密码）/ 临时邮箱 / API 安全 / 自动化；顶部水平 Tab 栏，纯文字
  - Provider 切换改为单选按钮，GPTMail/CF Worker 两套配置面板独立显示/隐藏
  - CF Worker 域名同步后只读展示，不允许手动修改
  - **数据模型决策（2026-04-04 v0.3 完结）**：GPTMail 继续用 `temp_mail_domains`/`temp_mail_default_domain`/`temp_mail_prefix_rules`；CF Worker 新增独立 key `cf_worker_domains`/`cf_worker_default_domain`/`cf_worker_prefix_rules`，三对 key 互不覆盖
  - **Tab 切换自动保存**（密码字段除外），页面宽度不再限制 720px
  - **涉及文件**：`templates/index.html`（整体重写）、`static/js/main.js`（Tab 切换 + 自动保存 + 面板显隐）、`static/css/main.css`（Tab 样式 + 只读字段）、settings model 新增 2 个 key
- [ ] **JSON 结构化日志** — 引入 `python-json-logger`，将日志输出为 JSON 格式，便于 ELK/Loki 查询
- [ ] **OpenAPI / Swagger 文档** — 为外部 API（十几个端点）生成标准化 API 文档
- [ ] **CONTRIBUTING.md** — 添加贡献指南，规范开发流程和 PR 规范
- [ ] **CHANGELOG.md** — 引入 changelog 管理工具（Towncrier / keep-a-changelog），记录版本变更

### 低优先级（长期优化）

 - [ ] **数据库迁移框架** — 当前手动版本管理（v19）够用，未来可考虑引入 Alembic 支持 rollback
- [ ] **配置启动校验** — 用 pydantic-settings 对 `.env` 做启动时校验
- [ ] **CORS 配置** — 添加显式 CORS 配置
- [ ] **错误监控集成** — 接入 Sentry 等工具
- [ ] **容器安全扫描** — CI 中集成 Trivy 扫描 Docker 镜像
- [ ] **Token 刷新并发化**（当账号数量 > 1000 时考虑，详见安全讨论记录）

## 2. 当前技术现状

### 2.1 验证码提取路径

两条独立实现，行为存在差异：

| 维度 | Web 端 (`controllers/emails.py`) | External API (`services/external_api.py`) |
|------|----------------------------------|-------------------------------------------|
| 代码量 | ~600 行（Outlook 路径） | ~130 行 |
| 渠道遍历 | 硬编码 Graph×2 + IMAP×2 + preferred 快路径 | `build_verification_channel_plan()` 循环 |
| confidence gate | ❌ 未调用 | ✅ `apply_confidence_gate()` |
| mutual exclusion | `True`（默认值） | `False` |
| filter 能力 | 无 | `filter_messages()` |
| email_obj 构造 | 内联，3 处各不相同 | `_build_email_obj_from_detail()` |
| refresh_token 轮换 | 内联 DB 操作 | 通过 channel service |

### 2.2 IMAP 服务

- `get_emails_imap_with_server()`: 154 行，逐封 FETCH，每次调用建立独立连接
- `get_email_detail_imap_with_server()`: 325 行，独立连接获取单封详情
- Token 获取 `get_access_token_imap_result()`: 每次请求 MS endpoint，无缓存

### 2.3 Graph 服务

- `get_access_token_graph_result()`: 不返回 `scope` 字段
- `get_emails_graph()`: 无权限预检，无 `Mail.Read` 时直接发 API 请求收 401

### 2.4 通道路由

- `verification_channel_routing.py`: 183 行，`fetch_emails_for_channel()` / `fetch_email_detail_for_channel()` 分离
- 无缓存层，每次遍历 channel_plan 时所有渠道都会尝试

## 3. 核心技术决策

### 3.1 决策一：统一函数放置位置

**选定方案**：放在 `services/verification_channel_routing.py`

**理由**：

1. 该模块已承载渠道遍历逻辑（`build_verification_channel_plan`、`fetch_emails_for_channel` 等），是自然扩展点
2. 避免新建文件增加认知负担
3. controller 和 external_api 都已导入该模块

### 3.2 决策二：连接复用的封装粒度

**选定方案**：在 `services/imap.py` 新增 `fetch_and_detail_imap_with_server()` 组合函数

**理由**：

1. 连接管理（SSL 握手 + 认证 + folder select）是 imap.py 的职责
2. 组合函数可以被 routing 层直接调用，也可独立用于未来邮件列表优化
3. 不改动现有 `get_emails_imap_with_server()`，降低回归风险

**备选（弃用）**：修改现有函数增加 `include_detail` 参数 → 参数膨胀，破坏单一职责

### 3.3 决策三：Token 缓存的作用域

**选定方案**：进程级 `dict` + `threading.Lock`

**理由**：

1. 当前部署现实仍长期停留在 Gunicorn `-w 1` 单 worker，因此进程级缓存**暂时**可视为全局缓存
2. 无需 Redis / memcached 等外部依赖
3. `threading.Lock` 保护并发安全（APScheduler 后台线程 + Flask 请求线程）

> 2026-05-22 补注（Issue #69）：
> 上述“单 worker = 全局缓存”只是一条**当前部署约束下的临时事实**，不能继续上升为长期架构前提。
> 现网性能问题已经表明：单 sync worker 会显著放大首屏排队成本。
> 若未来推进 scheduler 解耦并允许多 worker，这里的进程级缓存方案也需要重新评估一致性边界。

### 3.4 决策四：通道缓存独立模块

**选定方案**：新建 `services/channel_capability_cache.py`

**理由**：

1. 与 token 缓存职责分离（token cache 缓存凭据，channel cache 缓存能力状态）
2. 独立模块方便 mock 和测试
3. 未来可能升级为 DB 持久化或 Redis 缓存，隔离变更影响

### 3.5 决策五：批量 FETCH 的解析策略

**选定方案**：保留逐封 FETCH 但使用批量 ID

```python
# 原有（N 次网络往返）
for msg_id in paged_ids:
    status, msg_data = connection.fetch(msg_id, "(RFC822)")

# 优化后（1 次网络往返）
ids_str = b",".join(paged_ids)
status, all_data = connection.fetch(ids_str, "(RFC822)")
# 解析 all_data 中的交错数据
```

**理由**：imaplib 原生支持逗号分隔 ID 批量 FETCH，不需要修改库

### 3.6 决策六：并发双服务器的取消策略

**选定方案**：`ThreadPoolExecutor(max_workers=2)` + `as_completed()` + 不主动取消慢线程

**理由**：

1. `get_emails_imap_with_server` 内部的 IMAP 连接不支持线程间取消（imaplib 非线程安全）
2. 竞速获取第一个成功结果后，另一个线程自然完成并释放连接（finally logout）
3. 最多浪费一次 IMAP 连接，但不会泄漏资源

```python
def extract_verification_for_outlook(...):
    preferred = normalize_verification_channel(account.get("preferred_verification_channel"))
    channel_plan = build_verification_channel_plan(preferred)
    
    # 通道缓存过滤
    channel_plan = channel_capability_cache.filter_channel_plan(account["email"], channel_plan)
    
    # Graph 权限预检
    graph_scope = _check_graph_scope_if_needed(account, proxy_url)
    if graph_scope is not None and not has_mail_read_permission(graph_scope):
        channel_plan = [ch for ch in channel_plan if not ch.startswith("graph_")]
    
    any_channel_read_success = False
    upstream_errors = {}
    last_extracted = None
    new_refresh_token = None
    
    for channel in channel_plan:
        # fetch emails（IMAP 渠道使用连接复用版本）
        if channel.startswith("imap_"):
            channel_result = _fetch_and_detail_for_imap_channel(account, channel, proxy_url, top=20)
        else:
            channel_result = fetch_emails_for_channel(account=account, channel=channel, proxy_url=proxy_url, top=20)
        
        if not channel_result.get("success"):
            upstream_errors[channel] = channel_result.get("error")
            channel_capability_cache.set_status(account["email"], channel, available=False)
            continue
        
        any_channel_read_success = True
        channel_capability_cache.set_status(account["email"], channel, available=True)
        
        # refresh_token 轮换
        if channel_result.get("new_refresh_token"):
            new_refresh_token = channel_result["new_refresh_token"]
        
        emails = channel_result.get("emails", [])
        
        # 过滤（External API 场景）
        if from_contains or subject_contains or since_minutes or baseline_timestamp:
            emails = filter_messages(emails, from_contains, subject_contains, since_minutes, baseline_timestamp)
        
        if not emails:
            continue
        
        emails.sort(key=lambda x: ..., reverse=True)
        latest = emails[0]
        
        # 获取详情（IMAP 已在上面一步完成，Graph 需要单独获取）
        if channel.startswith("imap_"):
            detail = channel_result.get("detail")  # 连接复用已包含
        else:
            detail = fetch_email_detail_for_channel(account=account, channel=channel, message_id=latest["id"], proxy_url=proxy_url)
        
        if not detail:
            continue
        
        # 构造 email_obj → 提取 → AI 增强 → confidence gate
        email_obj = _build_email_obj(detail, latest)
        extracted = extract_verification_info_with_options(email_obj, ...)
        extracted = enhance_verification_with_ai_fallback(email_obj, extracted, ...)
        extracted = apply_confidence_gate(extracted, enforce_mutual_exclusion=False)
        
        # 填充元数据
        extracted.update({"email": ..., "matched_email_id": ..., "from": ..., ...})
        last_extracted = extracted
        
        # 判断是否命中
        if _should_return(extracted, expected_field):
            # 更新 preferred_channel
            accounts_repo.update_preferred_verification_channel(account["id"], channel)
            return {"success": True, "data": extracted, "channel_used": channel, "new_refresh_token": new_refresh_token}
    
    # 遍历完毕
    if last_extracted:
        return {"success": True, "data": last_extracted, ...}
    
    if not any_channel_read_success:
        return {"success": False, "error_code": "ACCOUNT_AUTH_EXPIRED", ...}
    
    return {"success": False, "error_code": "EMAIL_NOT_FOUND", ...}
```

```python
def fetch_and_detail_imap_with_server(
    account: str,
    client_id: str,
    refresh_token: str,
    folder: str = "inbox",
    skip: int = 0,
    top: int = 1,
    server: str = IMAP_SERVER_NEW,
) -> Dict[str, Any]:
    """一次 IMAP 连接完成邮件列表 + 最新一封详情"""
    token_result = get_access_token_imap_result(client_id, refresh_token)
    if not token_result.get("success"):
        return {"success": False, "error": token_result.get("error"), "emails": [], "detail": None}

    access_token = token_result["access_token"]
    connection = None
    try:
        connection = imaplib.IMAP4_SSL(server, IMAP_PORT)
        auth_string = f"user={account}\1auth=Bearer {access_token}\1\1".encode("utf-8")
        connection.authenticate("XOAUTH2", lambda x: auth_string)
        
        # SELECT folder（复用现有 folder_map 逻辑）
        selected_folder = _select_folder(connection, folder)
        if not selected_folder:
            return {"success": False, "error": ..., "emails": [], "detail": None}
        
        # SEARCH
        status, messages = connection.search(None, "ALL")
        if status != "OK" or not messages or not messages[0]:
            return {"success": True, "emails": [], "detail": None}
        
        message_ids = messages[0].split()
        total = len(message_ids)
        start_idx = max(0, total - skip - top)
        end_idx = total - skip
        if start_idx >= end_idx:
            return {"success": True, "emails": [], "detail": None}
        
        paged_ids = message_ids[start_idx:end_idx][::-1]
        
        # FETCH 摘要（同一连接内）
        emails_data = []
        detail = None
        latest_raw = None
        
        for i, msg_id in enumerate(paged_ids):
            status, msg_data = connection.fetch(msg_id, "(RFC822)")
            if status != "OK" or not msg_data or not msg_data[0]:
                continue
            raw_email = msg_data[0][1]
            msg = email.message_from_bytes(raw_email)
            
            body_preview = get_email_body(msg)
            emails_data.append({
                "id": msg_id.decode() if isinstance(msg_id, bytes) else str(msg_id),
                "subject": decode_header_value(msg.get("Subject", "无主题")),
                "from": decode_header_value(msg.get("From", "未知发件人")),
                "date": msg.get("Date", "未知时间"),
                "body_preview": body_preview[:200] + "..." if len(body_preview) > 200 else body_preview,
            })
            
            # 第一封（最新）同时解析为 detail
            if i == 0:
                raw_text = raw_email.decode("utf-8", errors="replace") if isinstance(raw_email, (bytes, bytearray)) else ""
                detail = {
                    "id": emails_data[-1]["id"],
                    "subject": emails_data[-1]["subject"],
                    "from": emails_data[-1]["from"],
                    "to": decode_header_value(msg.get("To", "")),
                    "cc": decode_header_value(msg.get("Cc", "")),
                    "date": emails_data[-1]["date"],
                    "body": get_email_body(msg),
                    "body_html": _get_html_body(msg),
                    "raw_content": raw_text,
                }
        
        return {"success": True, "emails": emails_data, "detail": detail}
    except Exception as exc:
        return {
            "success": False,
            "error": build_error_payload("EMAIL_FETCH_FAILED", "获取邮件失败", type(exc).__name__, 500, str(exc)),
            "emails": [],
            "detail": None,
        }
    finally:
        if connection:
            try:
                connection.logout()
            except Exception:
                pass
```

```python
def fetch_emails_and_detail_for_channel(
    *,
    account: Dict[str, Any],
    channel: str,
    proxy_url: str = "",
    skip: int = 0,
    top: int = 20,
) -> Dict[str, Any]:
    """组合 fetch，IMAP 渠道返回 emails + detail，Graph 渠道只返回 emails"""
    normalized = normalize_verification_channel(channel)
    
    if normalized in (CHANNEL_GRAPH_INBOX, CHANNEL_GRAPH_JUNK):
        # Graph 无连接复用收益，复用现有 fetch
        return fetch_emails_for_channel(account=account, channel=channel, proxy_url=proxy_url, skip=skip, top=top)
    
    # IMAP 渠道使用连接复用
    imap_server = IMAP_SERVER_NEW if normalized == CHANNEL_IMAP_NEW else IMAP_SERVER_OLD
    result = imap_service.fetch_and_detail_imap_with_server(
        str(account.get("email") or ""),
        str(account.get("client_id") or ""),
        str(account.get("refresh_token") or ""),
        folder="inbox",
        skip=int(skip or 0),
        top=int(top or 20),
        server=imap_server,
    )
    
    if not result.get("success"):
        return {"success": False, "error": result.get("error"), "channel": normalized}
    
    # 为 emails 添加 channel 标记
    emails = []
    for item in result.get("emails", []):
        enriched = dict(item)
        enriched["folder"] = "inbox"
        enriched["_verification_channel"] = normalized
        emails.append(enriched)
    
    return {
        "success": True,
        "emails": emails,
        "detail": result.get("detail"),  # IMAP 连接复用的核心收益
        "channel": normalized,
    }
```

### 4.3 IMAP OAuth Token 缓存

#### 4.3.1 缓存实现

文件：`outlook_web/services/imap.py`

```python
import hashlib
import threading
import time

_token_cache: Dict[str, tuple] = {}  # key → (access_token, expires_at)
_token_cache_lock = threading.Lock()


def _make_cache_key(client_id: str, refresh_token: str) -> str:
    """构造缓存 key：client_id + refresh_token 的短 hash"""
    rt_hash = hashlib.sha256(refresh_token.encode("utf-8")).hexdigest()[:16]
    return f"{client_id}:{rt_hash}"


def clear_imap_token_cache(client_id: str = None) -> None:
    """清除 token 缓存。client_id=None 时清除全部。"""
    with _token_cache_lock:
        if client_id is None:
            _token_cache.clear()
        else:
            keys_to_remove = [k for k in _token_cache if k.startswith(f"{client_id}:")]
            for k in keys_to_remove:
                del _token_cache[k]
```

#### 4.3.2 集成到 `get_access_token_imap_result()`

```python
def get_access_token_imap_result(client_id: str, refresh_token: str) -> Dict[str, Any]:
    cache_key = _make_cache_key(client_id, refresh_token)
    
    # 读缓存
    with _token_cache_lock:
        cached = _token_cache.get(cache_key)
        if cached:
            access_token, expires_at = cached
            if time.monotonic() < expires_at:
                return {"success": True, "access_token": access_token}
    
    # 缓存未命中 → 请求 MS endpoint（原有逻辑）
    try:
        res = requests.post(TOKEN_URL_IMAP, data={...}, timeout=30)
        if res.status_code != 200:
            return {"success": False, "error": ...}
        
        payload = res.json()
        access_token = payload.get("access_token")
        if not access_token:
            return {"success": False, "error": ...}
        
        # 写缓存
        expires_in = int(payload.get("expires_in", 3599))
        ttl = max(0, expires_in - 60)  # 提前 60s 过期
        with _token_cache_lock:
            _token_cache[cache_key] = (access_token, time.monotonic() + ttl)
        
        return {"success": True, "access_token": access_token}
    except Exception as exc:
        return {"success": False, "error": ...}
```

#### 4.3.3 key 安全性

- `refresh_token` 不直接用作 key（敏感数据），使用 SHA-256 前 16 位 hex
- 碰撞概率：16 hex = 64 bit → 生日悖论 ~2^32 个不同 token 才可能碰撞，远超实际规模
- `client_id` 明文作为前缀，无敏感性

```python
def has_mail_read_permission(scope: Any) -> bool:
    """检查 Graph API scope 是否包含邮件读取权限"""
    scope_str = str(scope or "")
    return "Mail.Read" in scope_str or "Mail.ReadWrite" in scope_str
```

### 4.5 通道缓存

#### 4.5.1 模块设计

新建文件：`outlook_web/services/channel_capability_cache.py`

```python
"""
通道能力缓存 — 记忆每个账号各渠道的可用/不可用状态。
进程级内存缓存，TTL 1 小时。
"""
from __future__ import annotations

import threading
import time
from typing import Any, Dict, List, Optional

_DEFAULT_TTL = 3600  # 1 小时

# 数据结构：{email: {channel: {"status": "available"|"unavailable", "expires_at": float}}}
_cache: Dict[str, Dict[str, Dict[str, Any]]] = {}
_lock = threading.Lock()


def set_status(email: str, channel: str, *, available: bool) -> None:
    """记录渠道可用性"""
    with _lock:
        if email not in _cache:
            _cache[email] = {}
        _cache[email][channel] = {
            "status": "available" if available else "unavailable",
            "expires_at": time.monotonic() + _DEFAULT_TTL,
        }


def get_status(email: str, channel: str) -> Optional[str]:
    """查询渠道状态。返回 "available"/"unavailable" 或 None（未知）"""
    with _lock:
        account_cache = _cache.get(email)
        if not account_cache:
            return None
        entry = account_cache.get(channel)
        if not entry:
            return None
        if time.monotonic() >= entry["expires_at"]:
            del account_cache[channel]
            return None
        return entry["status"]


def filter_channel_plan(email: str, channel_plan: List[str]) -> List[str]:
    """过滤掉已知不可用的渠道"""
    return [ch for ch in channel_plan if get_status(email, ch) != "unavailable"]


def clear_for_account(email: str) -> None:
    """清除指定账号的全部缓存（token 刷新时调用）"""
    with _lock:
        _cache.pop(email, None)


def clear_all() -> None:
    """清除全部缓存（测试用）"""
    with _lock:
        _cache.clear()
```

### 4.6 IMAP 批量 FETCH

#### 4.6.1 解析策略

imaplib `connection.fetch(b"40,41,42", "(RFC822)")` 返回的 `all_data` 结构：

```python
[
    ((b'42 (RFC822 {12345}', b'...raw email bytes...'), b')'),
    ((b'41 (RFC822 {11111}', b'...raw email bytes...'), b')'),
    ((b'40 (RFC822 {22222}', b'...raw email bytes...'), b')'),
]
```

交错出现 `(header_bytes, body_bytes)` tuple 和 `b")"` 闭合标记。

#### 4.6.2 解析实现

```python
def _parse_batch_fetch_response(all_data: list) -> List[tuple]:
    """从批量 FETCH 响应中提取 (msg_id, raw_bytes) 列表"""
    results = []
    for item in all_data:
        if isinstance(item, tuple) and len(item) == 2:
            header, raw_email = item
            if isinstance(raw_email, bytes) and isinstance(header, bytes):
                # 从 header 中提取 msg_id：b'42 (RFC822 {12345}'
                msg_id_str = header.split(b" ")[0].decode("ascii", errors="ignore")
                results.append((msg_id_str, raw_email))
    return results
```

### 4.7 IMAP 并发双服务器

#### 4.7.1 实现

文件：`outlook_web/services/imap.py`

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def get_emails_imap_concurrent(
    account: str,
    client_id: str,
    refresh_token: str,
    folder: str = "inbox",
    skip: int = 0,
    top: int = 20,
    servers: tuple = (IMAP_SERVER_NEW, "outlook.office365.com"),
) -> Dict[str, Any]:
    """并发连接多台 IMAP 服务器，返回第一个成功结果"""
    if len(servers) <= 1:
        server = servers[0] if servers else IMAP_SERVER_NEW
        return get_emails_imap_with_server(account, client_id, refresh_token, folder, skip, top, server)
    
    last_error = None
    with ThreadPoolExecutor(max_workers=len(servers)) as executor:
        futures = {
            executor.submit(
                get_emails_imap_with_server,
                account, client_id, refresh_token, folder, skip, top, server
            ): server
            for server in servers
        }
        
        for future in as_completed(futures):
            result = future.result()
            if result.get("success"):
                return result
            last_error = result
    
    return last_error or {"success": False, "error": {"code": "ALL_SERVERS_FAILED", "message": "所有服务器连接失败"}}
```

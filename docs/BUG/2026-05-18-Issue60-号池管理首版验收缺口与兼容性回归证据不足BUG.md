 ## 1. 问题概述

Issue #60 的首版实现已完成 MVP 主链路（池内/池外、移入、移出、`claimed` 保护），但在验收对照中发现两类缺口：

1. **功能口径缺口**：前端筛选未提供 `group` 入口（FD/TODO 口径要求有 group 维度筛选）
2. **回归证据缺口**：尚缺“旧接口未受影响”的自动化回归证明闭环

该问题不是“核心功能不可用”，而是“**文档承诺与上线验证闭环不完整**”。

> 2026-05-18 更新：本 BUG 对应的方案 A 修复已完成，见第 8 节“修复落地结果”。

 ---

 ## 2. 现场证据

 ### 2.1 group 筛选缺口

1. `templates/index.html` 的 `page-pool-admin` 筛选栏当前包含：
   - `in_pool`
   - `pool_status`
   - `provider`
   - `search`
2. `static/js/features/pool_admin.js` 查询参数当前仅发送：
   - `in_pool`
   - `pool_status`
   - `provider`
   - `search`
   - `page` / `page_size`
3. 后端 `outlook_web/repositories/pool_admin.py` 已支持 `group_id` 过滤，但前端未接入对应控件与参数。

 ### 2.2 兼容性回归证据缺口

本轮已执行：

`python -m unittest tests.test_pool_admin_repository tests.test_pool_admin_service tests.test_pool_admin_api tests.test_pool_admin_frontend_contract -v`

结果：`Ran 40 tests ... OK`。

但上述仅覆盖新增号池管理链路，尚未形成以下“旧链路无副作用”的自动化证明：

1. `GET /api/accounts` 旧语义回归
2. `/api/external/pool/*` 旧契约回归
3. overview 邮箱池统计链路回归

 ---

 ## 3. 影响范围

 ### 3.1 直接影响

1. 管理员无法在号池页按分组快速筛选，运营效率低于 FD 预期。
2. 发布评审阶段缺少“旧接口兼容已验证”的客观证据，影响上线信心与变更审批。

 ### 3.2 间接影响

1. 文档与实现出现细小偏差，后续多 agent 协作容易反复争议“是否达标”。
2. 若后续出现旧接口问题，缺少本阶段回归基线可追溯。

 ### 3.3 不影响

1. `claimed` 保护主逻辑可用。
2. `move_into_pool / move_out_of_pool` 主动作可用。
3. 新增 `pool-admin` 核心接口可用。

 ---

 ## 4. 根因分析

1. **范围收敛导致的前端欠账**：首版实现优先闭环 MVP，前端筛选只落了高频项，遗漏了 group 控件接入。
2. **测试分层偏新增链路**：当前新增测试覆盖了新模块自身正确性，但未把“旧接口兼容回归”并入同一轮执行清单。
3. **阶段状态表达不充分**：此前文档强调“已实现待验收”，但对“哪些验收点仍未闭合”标注不够前置，导致审查时暴露。

 ---

 ## 5. 修复与设计建议

 ### 方案 A（推荐）：补齐实现与回归证据

1. 前端新增 group 筛选控件并透传 `group_id`
2. 增补并执行兼容回归测试：
   - `GET /api/accounts`
   - `/api/external/pool/*`
   - overview pool 统计接口
3. 在 TODO / FD / TDD 更新“已闭合”状态

**优点**：保持既有文档口径不变，交付一致性最高。  
**成本**：需要额外 1 轮实现+回归。

 ### 方案 B（备选）：降级口径，暂不补实现

1. 明确把 group 筛选降级为增强项
2. 将兼容回归作为上线前 checklist，而非本阶段必须执行

**优点**：最快结束当前阶段。  
**风险**：文档会收缩，且上线证据链较弱。

 ---

 ## 6. 建议验收标准（修复后）

1. 号池页可按 group 过滤（UI + API 参数 + 查询结果一致）
2. `GET /api/accounts` 回归通过
3. `/api/external/pool/*` 回归通过
4. overview pool 统计回归通过
5. WORKSPACE 与 TODO 同步更新闭环记录

 ---

 ## 7. 当前建议结论

建议选择 **方案 A**：补齐 group 筛选与兼容性回归证据，再进入“完整通过”状态。
